# Changelog

## v0.2.0 - Polish Pass After Initial Commit

这一版是在初始提交 `小游戏初始版本` 之后，围绕“更好玩、更像完整作品、更适合手机安装试玩”连续打磨出来的版本。

### 核心玩法与节奏

- 将玩法稳定为三轨闪避，降低早期挫败感，让玩家有真实反应时间。
- 调整障碍生成逻辑，避免同一时刻多个轨道同时刷出不可读局面。
- 优化障碍间距与刷新节奏，让游戏不再“第一关都过不去”，同时保留上头感。
- 新增 `FLOW / FEVER` 节奏循环，并反复调平视觉提示，最终保留速度感与轻微震动提示，去掉不舒服的长期全屏变色。
- 将障碍尺寸改为随画布宽度轻微自适应，手机端视觉更轻盈，电脑端观感基本稳定。

### 视觉与界面

- 重做游戏内 HUD，把节奏槽、分数、最佳整合到同一条顶部信息带中。
- 多轮调整 HUD 比例、排版和文案，减少遮挡，提升 1080p、2K 和手机长屏下的可读性。
- 新增极简三角暂停按钮，并反复微调其位置，避免与 HUD、摄像头区域或其他图形重叠。
- 精简首页和结算页文案，把品牌信息收纳进开始卡片与结算卡片，减少屏幕占用。
- 增加独立的“赞赏作者”页面，避免首页信息过载。

### 交互与状态管理

- 新增暂停功能，并补齐暂停覆盖层。
- 为网页端增加 `ESC` 返回逻辑，为 Android 端增加系统返回键逻辑。
- 将返回键行为细化为“优先返回上一层，再回主页”：
  - 赞赏页先关闭赞赏页
  - 暂停页先恢复游戏
  - 结算页返回主页
  - 游戏中可直接返回主页
- 修复赞赏页打开时游戏仍在后台继续运行的问题，改为进入赞赏页自动暂停，退出后恢复原状态。

### 音乐与反馈

- 增加内置 Web Audio 背景音乐与音效。
- 多次调整背景音乐风格，使其更轻、更温柔，不抢玩法注意力。
- 保留切换、擦边、连击、碰撞、FEVER 触发等关键反馈音，让节奏感更完整。

### 手机适配与 Android 封装

- 从纯网页版本扩展到 PWA 与 Android WebView 双形态。
- 修正 Android 状态栏与底部导航栏沉浸式带来的遮挡问题，让系统栏保留但颜色和游戏背景统一。
- 多轮修正手机长屏适配策略，最终改为根据真实可用区域重算画布，而不是简单固定比例硬撑满。
- 同步维护根目录网页资源与 `android-app/app/src/main/assets/www/` 中的 Android 资源，确保网页试玩与 APK 行为一致。

### 工程化与项目管理

- 从丢失大部分文件的状态中，根据残留的 `android-app/` 工程反推出原始网页项目结构，并恢复缺失文件。
- 建立 `.gitignore`，忽略 Gradle 构建产物、本地 IDE 配置和私有赞赏码。
- 整理 README、恢复报告与 Git 使用说明，补齐项目文档。

### 面试可讲的点

- 如何从用户试玩反馈中不断收敛玩法难度与视觉表达。
- 如何处理“桌面端看起来没问题，但手机端出现长屏适配、系统栏遮挡、状态管理冲突”等真实问题。
- 如何在没有复杂框架的前提下，用原生 HTML/CSS/JS 做完整的游戏循环、音效、状态机和移动端适配。
- 如何把一个轻量网页小游戏进一步包装成可安装的 Android 应用。
